Forms

There are many ways to style forms and input elements. The main input elements are buttons, textboxes, tetareas, alerts and select menus.

Buttons

These can be styled in multiple ways to change their importance to the user or the action that they can complete. They should be used for actions as much as possible. The base button is a button element with the .btn class applied. Each button also has 3 sizes: normal which has no class, .mid and .small. By default the button's color is WOLAS Blue. You can also apply the button classes to links.

<button class="btn">Button</button><button class="btn mid">Button .mid</button><button class="btn small">Button .small</button><br/>
<button class="btn warn">Button</button><button class="btn warn mid">Button .mid </button><button class="btn warn small">Button .small</button><br/>
<button class="btn erro">Button</button><button class="btn erro mid">Button .mid </button><button class="btn erro small">Button .small</button><br/>
<button class="btn succ">Button</button><button class="btn succ mid">Button .mid </button><button class="btn succ small">Button .small</button><br/>
<button class="btn plain">Button</button><button class="btn plain mid">Button .mid </button><button class="btn plain small">Button .small</button><br/>

Text Boxes

Text boxes are created using the input tag with the type="text" specification. All inputs have padding already applied to them. A blue box-shadow effect is applied when the input is focused. You can also apply the focus effect yourself by using the .focus class. For a watermark, use the placeholder property on the tag.

<input type="text" /> <input type="text" placeholder="This is a watermark"/>  <input type="text" class="focus" placeholder="Textbox with .focus"/>

If there is an error with user input or there is a required field, you can apply the .error class to the element.

<input type="text" class="error" placeholder="Textbox with .error" />

Textarea

All of the above classes etc. also apply to the textarea element.

<textarea rows="10" cols="50"></textarea>

Dropdown

Dropdown lists use the select element and don't have any specific style modifiers apart from the .focus and .error classes.

<select>
	<option>Please Select</option>
	<option>Martin Brennan</option>
	<option>Steve Bennet</option>
</select>
<select class="error">
	<option>.error</option>
	<option>Martin Brennan</option>
	<option>Steve Bennet</option>
</select>
<select class="focus">
	<option>.focus</option>
	<option>Martin Brennan</option>
	<option>Steve Bennet</option>
</select>

Checkboxes and Radio Buttons

These elements don't use any specific styles. A checkbox uses the input tag with the type="checkbox" specification. A radio button uses the input tag with the type="radio" specification.

<input type="checkbox">This is a checkbox.</input>
<input type="radio">This is a radio.</input>
This is a checkbox. This is a radio.

Disabled and Readonly Inputs

All inputs can have the disabled or readonly property added to their tag to grey out the input and prevent anyone from using it.

<input type="checkbox" disabled>This is a checkbox.</input>
<input type="radio" disabled>This is a radio.</input>
<input type="text" disabled />
<select disabled>
	<option>Please Select</option>
	<option>Martin Brennan</option>
	<option>Steve Bennet</option>
</select><br/>
<textarea rows="10" cols="50" disabled></textarea>
This is a checkbox. This is a radio.

Alerts

There are four different types of alert message that can be used to tell the user important information, warn them about an action or tell them about successes or failures. An alert uses the div tag, with the .a class applied. Each different alert has its own class too; .a-info for a blue information alert, .a-warn for a yellow warning alert, .a-erro for an error and .a-succ for a successful action.

<div class="a a-info">This is an informative and helpful message.</div>
<div class="a a-warn">Are you sure? That could do something bad!</div>
<div class="a a-erro">Something has gone terribly wrong!</div>
<div class="a a-succ">Great! Everything went smoothly and the action is complete.</div>
